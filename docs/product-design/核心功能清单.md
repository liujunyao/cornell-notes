# 二、产品原型草图说明（核心页面+交互标注）

原型草图聚焦“核心页面布局+关键交互路径”，标注各元素位置、功能映射及优先级，适配主流设计工具（Figma/Sketch），开发可直接参考落地。以下按页面类型说明，草图框架统一采用“移动端（手机/平板）+PC端”双适配。

## 1. 核心页面原型框架（按使用频次排序）

### 1.1 首页（所有角色通用，P0）

* 布局框架：
  * 顶部：搜索框（占宽80%）+ 快捷新建按钮（图标式，右侧固定）；
  * 中部：三大快捷入口（最近编辑/待复习/收藏笔记），卡片式展示，点击直接进入；下方为笔记分类列表（按创建时间倒序，显示分类名称、笔记标题、最后编辑时间）；
  * 底部：导航栏（首页/复习/协作/我的，4个图标+文字，当前页面标红高亮）。
* 交互标注：
  * 搜索框：输入关键词实时联想，显示匹配结果（含笔记标题/内容预览），点击结果进入笔记详情；
  * 快捷新建：点击直接跳转至三分栏编辑页，默认生成空白笔记，标题为“未命名笔记+时间戳”；
  * 分类列表：长按笔记卡片弹出操作菜单（删除/分享/标星），点击卡片进入笔记编辑页。
* 多端适配：
  * PC端：中部快捷入口横向排列，分类列表采用网格视图（可切换列表/网格），右侧可显示“复习计划日历”；
  * 手机端：中部快捷入口纵向排列，分类列表仅显示核心信息（标题+时间），节省空间。

### 1.2 笔记编辑页（核心页面，P0）

* 布局框架（三分栏核心）：
  * 顶部：笔记标题输入框（居中，可修改）+ 操作按钮（左侧：保存/返回；右侧：AI功能/分享/更多）；
  * 中部：左侧线索栏（25%）、右侧笔记栏（75%），两栏顶部均有轻量工具栏（悬浮显示）；
  * 底部：总结栏（跨左右两栏，高度20%），顶部显示“总结”字样，支持工具栏操作。
* 交互标注：
  * 工具栏：鼠标悬停显示，含基础编辑（加粗/列表）、格式插入（图片/公式）、高亮、AI功能（仅笔记栏/线索栏/总结栏对应位置显示）；
  * AI功能按钮：笔记栏显示“AI自动高亮/AI拓展”，线索栏显示“AI提炼”，总结栏显示“AI生成”，点击后显示加载状态（转圈图标），生成后展示候选内容；
  * 栏调整：鼠标拖拽两栏边界可调整宽度，释放后自动保存比例，移动端支持手势切换焦点栏。
* 多端适配：
  * 平板横屏：线索栏可设为抽屉（点击左侧边缘展开），最大化笔记栏视野，总结栏固定底部；
  * 手机端：竖屏隐藏线索栏/总结栏，顶部显示“线索”标签（下拉展开），底部显示“总结”标签（上滑展开），默认全屏显示笔记栏。

### 1.3 复习模式页（P0基础，P1进阶）

* 布局框架：
  * 顶部：笔记标题 + 模式切换按钮（基础/进阶/高阶，单选，当前模式高亮）+ 退出按钮；
  * 中部：左侧线索栏（正常显示），右侧笔记栏（基础/高阶模式遮盖，进阶模式显示提问框）；
  * 底部：操作按钮（上一行/下一行/标记薄弱点/完成复习）。
* 交互标注：
  * 基础模式：笔记栏显示半透明遮盖层，点击“解锁一行”显示对应内容，全部解锁后提示“可开始复述验证”；
  * 进阶模式：笔记栏替换为问答框，AI显示问题，用户输入答案后点击“比对”，下方显示标准答案及正确率；
  * 完成复习：点击后弹出复习报告（时长/正确率/薄弱点），用户标记“已掌握/未掌握”，系统更新下次复习时间。

### 1.4 教师班级管理页（P1）

* 布局框架：
  * 顶部：班级名称 + 操作按钮（添加学生/分发模板/批量反馈）；
  * 中部：学生列表（显示姓名/账号/笔记提交状态）+ 笔记列表（按作业/课程分类）；
  * 底部：统计信息（已提交/未提交人数、平均复习时长）。
* 交互标注：
  * 添加学生：生成班级二维码/邀请码，支持批量导入学生账号；
  * 分发模板：选择已保存的笔记模板，设置提交截止时间，发布后学生端接收提醒；
  * 批改笔记：点击学生笔记进入详情页，添加批注（文字/标红），打分后提交反馈，学生端实时接收。

### 1.5 个人中心页（P0）

* 布局框架：
  * 顶部：用户头像/昵称/角色，下方为功能入口（我的笔记/复习计划/收藏/协作小组）；
  * 中部：设置入口（个性化/账号安全/数据备份/帮助反馈）；
  * 底部：版本信息/退出登录按钮。
* 交互标注：
  * 复习计划：点击进入日历视图，显示每日复习任务，点击任务直接进入对应笔记复习；
  * 数据备份：点击“导出备份”生成加密包，“导入备份”选择本地文件，导入前提示覆盖风险；
  * 退出登录：二次确认，退出后清除本地缓存（可选保留离线笔记）。

## 2. 原型草图标注规范（统一标准）

* 元素标注：所有按钮/输入框/导航栏需标注“功能名称+优先级+交互反馈”，例：【快捷新建按钮（P0）- 点击跳转至笔记编辑页，生成空白笔记】；
* 尺寸标注：PC端标注固定宽度（如搜索框宽度80%，快捷按钮尺寸40\*40px），移动端标注适配比例（如笔记栏占屏比90%）；
* 交互路径：用箭头标注核心操作流程，例：首页→快捷新建→笔记编辑页→保存→返回首页；
* 状态标注：标注元素不同状态（默认/hover/点击/加载），例：按钮默认灰色，hover蓝色，点击深色，加载时显示转圈图标。

## 3. 原型落地建议

* 工具选择：优先使用Figma，创建“移动端/平板端/PC端”三个文件，共用组件库（按钮/工具栏/导航栏），保证设计一致性；
* 优先级适配：P0功能全部绘制高保真原型，标注详细交互；P1功能绘制中保真原型，明确核心逻辑；P2功能仅绘制线框图，预留迭代空间；
* 开发对接：原型导出为PDF（含标注），同步导出Figma链接，标注各页面跳转关系，便于开发理解完整流程。

# 三、总结

本清单与原型说明严格对齐原有产品方案，聚焦“可开发落地”，核心功能清单明确了各模块优先级与交互逻辑，避免开发歧义；产品原型草图规范了页面框架、标注标准，覆盖全核心场景与多端适配。开发可按“P0→P1→P2”优先级逐步落地，先实现核心笔记与复习功能，再迭代AI辅助与协同功能，确保产品基础体验稳定后再做差异化创新。
